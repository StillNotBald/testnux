## Testing Log Convention

This folder follows the TrunkNuX three-discipline-track pattern:

| Track | Location | Purpose |
|---|---|---|
| Requirements | `requirements/REQUIREMENTS.md` | R-XX product requirements |
| Sprint log | `sprint-log/<date>_<feature>/` | What was built (decisions, retros) |
| Testing log | `testing-log/<date>_<slug>/` | What was tested (this folder) |

Each test-pass folder is **date-prefixed** so the testing-log/ index (testing-log/README.md)
stays sortable by session. Never remove old folders — they are audit evidence.
